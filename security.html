Security

While researching security vulnerabilities, it is requested that you refrain from:
  • Denial of service
  • Spamming
  • Social engineering (including phishing)
  • Any physical attempts against data centers
Please include the following in your report:
  • Description of the vulnerability
  • Steps to reproduce the vulnerability
  • Proof of exploitability
  • Impact
  • CVSSv3 Score
  • Proof of Concept
  • Browser, OS and/or app version used during testing
  • IP Address(es) from which testing was done
Please note that no monetary rewards will be given for security issues
Report security issues using the contact form at https://www.ndt3.me/#contact

In scope:

  • *.ndt3.me
  • *.ndt3.ml
  • *.ndtweb.cf
  • *.n-cdn.cf
  • *.ndt3.top
  • Client websites (See https://ndtweb.freshdesk.com/support/solutions/articles/44000285387)
  • Software

Out of Scope:

  • *.ugc.n-cdn.cf (User Generated Content CDN)
  • *.ugc.cdn.ndt3.top
  • status.ndt3.ml (Externally hosted status page)
  • uptimemonitor.ndt3.ml (Externally hosted status page)
  • status.[client website] (Externally hosted status pages for clients)
  • ndtweb.freshdesk.com (Externally hosted support system)
  • *.ndtweb.tk (At this time, only used as a redirect to ndtweb.cf)
  • *.ndt3-servers.ml (Previously used to have all servers under one domain; no longer used)
  • nodeupload.ndt3.me/nodeupload.ndt3.ml (Static docs site; almost no security risk)
  • Self XSS
  • Missing cookie flags
  • SSL/TLS Best Practices
  • Login/Logout/Unauthenticated CSRF
  • Incomplete/Missing SPF/DKIM
  • Clickjacking
  • Known vulnerabilities in libraries without proof of concept
